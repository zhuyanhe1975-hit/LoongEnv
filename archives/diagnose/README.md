# archives/diagnose

这里是中枢仓中的**诊断类共享归档区**。

它用于沉淀需要跨成员复盘的诊断结论、故障分析、问题回溯与可复用排障经验。

## 与旧目录的关系

- 原根目录 `diagnose/` 已进入迁移态
- 后续新的共享诊断沉淀应优先进入 `archives/diagnose/`

## 模板

- [诊断归档模板](./_template.md)

## 边界

- 这里归档共享诊断结论，不承载成员仓本地调试过程
- 4D+C 中的 `Diagnose` 若属于项目实现结构，应保留在成员仓或项目架构文档中
