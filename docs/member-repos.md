# 成员仓说明

本文说明成员仓命名、接入方式与共享回流约定；总体边界以 [中枢仓与成员仓边界说明](./hub-repo-boundaries.md) 为准。

## 成员标识与仓名

成员相关命名分为三层：

| 层级 | 规则 | 示例 |
| --- | --- | --- |
| 成员标识 | 姓名字母缩写 | `ZHU_YH` |
| 本地接入目录 | `repos/<成员标识>` | `repos/ZHU_YH` |
| 远程成员仓 | `LoongEnv-<成员标识>` | `LoongEnv-ZHU_YH` |

- `ZHAO_SZ`：赵世泽
- `WANG_CZ`：王成志
- `ZHU_YH`：朱延河

## 作用

成员仓用于各自独立推进开发、实验、整理阶段成果，再将共享结果同步回 `LoongEnv` 中枢仓。

## 本地接入与远程仓

- `repos/<成员标识>`：中枢仓中的本地接入占位目录
- `LoongEnv-<成员标识>`：组织中的远程成员仓命名示例

例如，`ZHU_YH` 同时对应：

- 本地接入占位：`repos/ZHU_YH`
- 远程成员仓：`LoongEnv-ZHU_YH`

三层名称是**映射关系**，不是同一个受控仓库实体。

## 共享原则

- 个人实现留在成员仓
- 团队结论回流到中枢仓
- 需要共享的设计、部署、诊断结果同步到中枢仓对应目录
