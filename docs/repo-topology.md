# 仓库拓扑

本文补充说明中枢仓、成员仓与本地占位目录之间的拓扑关系；职责边界以 [中枢仓与成员仓边界说明](./hub-repo-boundaries.md) 为准。

## Team Hub

当前仓库是团队共享中枢，保存跨成员可见的内容。

## Member Repos

成员仓相关命名按三层展开：

| 层级 | 说明 | 示例 |
| --- | --- | --- |
| 成员标识 | 用于协作、看板与归档的成员代号 | `ZHU_YH` |
| 本地接入目录 | 中枢仓中的本地占位路径 | `repos/ZHU_YH` |
| 远程成员仓 | 组织中的真实成员仓 | `LoongEnv-ZHU_YH` |

成员仓负责个人实现、实验与阶段性交付；中枢仓只保存适合共享的回流结果。

### 本地接入占位

当前已知远程成员仓示例：

## AI Roles

AI 成员统一挂在 `LoongEnv` 中枢仓的 GitHub Actions 下：

- `DOCS_AI`
- `SUPERVISOR_AI`
- `DEPLOY_AI`

成员仓只保留各自项目的实现与发布流水线，不再承担团队级 AI 入口说明。
